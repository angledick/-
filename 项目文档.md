## 1. 项目概述

### 1.1 项目定位

**避风港(SafeHarbor)**是一款面向跨境电商卖家的**OS级合规智能体平台**,旨在为出海企业提供7×24小时在线的合规顾问服务。系统围绕商品出海的完整生命周期,构建了从感知、检查、推荐、告知、交互到处理的六阶段闭环,帮助卖家在商品上架前识别合规风险、实时监控法规变化、主动推送风险预警,并通过飞书等渠道实现团队协同。

### 1.2 目标用户

| 用户类型 | 核心诉求 | 解决方案 |
|---------|---------|---------|
| Shopify独立站卖家 | 避免合规问题导致关店 | 自动合规检查+Shopify深度集成 |
| 跨境新手团队 | 缺乏专职合规顾问 | AI合规助手+规则库自动匹配 |
| 多市场运营企业 | 多法规体系统一管理 | 欧盟/美国/日韩全市场覆盖 |
| 供应链管理者 | 实时掌握风险动态 | 主动监控+飞书预警推送 |

### 1.3 核心价值

- **事前预防** → 上架前自动识别认证要求、HS编码、VAT合规状态
- **实时监控** → 持续追踪四大市场法规动态,第一时间推送预警
- **主动告知** → 高危事件自动通过飞书通知团队,关键信息不遗漏
- **智能对话** → 自然语言提问,获得专业合规解答

## 2. 核心能力

### 2.1 商品合规预检

系统自动识别商品类别,匹配目标市场的认证要求:

| 市场 | 认证类型 | 检查项 |
|------|---------|-------|
| 欧盟 | CE/RoHS/GPSR | 安全指令、环保要求、产品责任 |
| 美国 | FCC/FDA/CPSC | 电磁兼容、食品药品安全、消费品安全 |
| 日本 | PSE/JFRL | 电气安全、食品接触材料 |
| 韩国 | KC/KFDA | 电子电器安全、食品药品安全 |

### 2.2 法规变化实时监控

- **监控范围**: 欧盟指令/美国联邦法规/日本法令/韩国标准
- **更新频率**: 定时任务每日扫描+事件触发实时更新
- **推送机制**: 飞书群消息+应用内通知+WebSocket实时推送

### 2.3 Shopify深度集成

| 集成能力 | 实现方式 | 触发时机 |
|---------|---------|---------|
| 商品同步 | Admin REST API直连 | 商品上架/修改时 |
| 合规字段补足 | 三层自动补足体系 | 商品创建/更新时 |
| Webhook监听 | Shopify事件订阅 | 实时接收店铺动态 |
| AI Toolkit | Claude Agent SDK执行 | 业务操作统一技能化 |

### 2.4 飞书协同能力

- **消息监听**: lark-cli事件消费,实时接收群消息
- **风险预警**: 自动推送至指定飞书群(RISK_INTEL_FEISHU_CHAT_ID)
- **人工审批**: 危险事件支持人工决策闭环
- **双向通信**: Worker与飞书双向消息通道

### 2.5 智能对话助手

- **SSE流式输出**: 支持思考过程/规划/流式文本/操作卡片
- **多Agent协作**: Manager拆解任务,Worker隔离执行
- **知识库检索**: RAG增强生成,精准匹配法规条款
- **CLI命令执行**: 支持系统命令交互式执行

### 3.2 六阶段合规流

```
感知 → 检查 → 推荐 → 告知 → 交互 → 处理
(Sense)(Check)(Recommend)(Notify)(Interact)(Process)
```

| 阶段 | 触发方式 | 执行者 | 输出 |
|------|---------|--------|------|
| **感知** | 用户动作/外部API/定时调度 | EventBus+RuleEngine | 标准化EventRecord |
| **检查** | 事件路由到Skill/规则集 | Worker Agent+Skill | 检查结果+风险等级 |
| **推荐** | 检查结果匹配规则库 | Skill推荐器+规则引擎 | 操作建议列表 |
| **告知** | 推荐结果匹配通知规则 | 通知引擎+频道适配器 | 多端消息推送 |
| **交互** | 用户确认/修改/补充 | 对话引擎+飞书 | 执行指令 |
| **处理** | 确认指令编排执行 | Workflow+Skill流 | 操作结果+回写 |

### 3.3 事件分类体系

系统支持**8类事件体系**:

| 事件类型 | 说明 | 示例 |
|---------|------|------|
| `lifecycle` | 产品生命周期事件 | 商品上架/修改/下架 |
| `compliance` | 合规检查事件 | CE认证缺失/VAT未注册 |
| `certification` | 认证相关事件 | 证书到期/认证更新 |
| `order` | 订单相关事件 | 新订单/退款/物流异常 |
| `regulation` | 法规变更事件 | 欧盟新指令发布 |
| `risk_alert` | 风险预警事件 | 商品列入管制清单 |
| `user_action` | 用户操作事件 | 手动触发检查 |
| `system` | 系统内部事件 | 定时任务执行/健康检查 |

- **流式响应**: 支持SSE逐事件推送(thinking/plan/token/skill_start/skill_end/done)

- **环境变量管理**: 自动注入API密钥与模型配置
- **技能化改造**: 所有操作统一通过Skill注册执行

- **YAML/Markdown配置中心化**: 事件/Worker/规则全部配置化

## 6. 跨境合规全流程

### 6.1 外贸总流程(11个环节)

| 环节 | 合规检查重点 |
|------|-------------|
| 1. 选品/样品设计 | 外观专利风险、文化风险、管控风险、地区局势风险 |
| 2. 供应商审核 | 供应商资质审核、税务合规(无票vs有票) |
| 3. 合同签订 | 合同合规性、交付金额/交付点风险 |
| 4. 生产 | 质量控制、产量保障 |
| 5. 交货/境内物流 | EXW/FCA/DAP/DDP条款风险 |
| 6. 订舱 | 保险、时效、报价优化 |
| 7. 出口报关 | 申报信息、HS编码、单证一致性 |
| 8. 进口清关 | 14日内申报、电子报关单 |
| 9. 缴税 | 关税计算、出口退税 |
| 10. 境外物流 | 货物安全到达仓储/买家 |
| 11. 交付/退货/纠纷 | 消费者权益保护、退货政策 |

### 6.2 Shopify独立站专属流程

#### 环节1: 建站与基础环境搭建

**合规检查**:
- 账号风控合规: 真实企业信息,禁止频繁变动VPN
- 法律页面配置: 《隐私政策》《服务条款》《退换货政策》
- GDPR/CCPA合规: Cookie同意弹窗

#### 环节2: 选品与样品设计

**合规检查**:
- 知识产权(IP)风险: 外观专利、商标侵权
- 产品准入与认证: CE/FCC/FDA/PSE/KC
- 文化与局势风险: 宗教禁忌、制裁地区

#### 环节3: 商品上架与内容合规

**合规检查**:
- 广告法与营销合规: 禁止虚假宣传、绝对化用语
- 数据追踪合规: Facebook Pixel/GA4隐私法合规

#### 环节4: 支付与收款配置

**合规检查**:
- 反洗钱与KYC审核: 真实身份认证
- 资金链路透明化: 持牌第三方机构结汇

#### 环节5: 订单处理与境内物流

**合规检查**:
- 三单一致性: 订单/支付/物流信息匹配

#### 环节6: 出口报关

**合规检查**:
- 9610清单核放: 跨境电商B2C直邮报关
- HS编码准确性: 申报品名/价值核对

#### 环节7: 进口清关与境外派送

**合规检查**:
- 关税与IOSS/VAT: 欧洲市场预收VAT
- 目的国税务合规: 销售税起征点

#### 环节8: 交付、售后与退货

**合规检查**:
- 消费者权益保护: 欧盟14天无理由退货
- 数据删除请求(DSAR): GDPR合规处理

#### 环节9: 财务结算与税务申报

**合规检查**:
- 境外收入申报: 如实申报,禁止私人账户隐匿
- 出口退税/免税申报: 定期向税务局申请

### 6.3 合规清单(6大维度)

| 维度 | 核心检查项 |
|------|-----------|
| **平台与账号** | 账号真实性、法律页面、SSL加密 |
| **数据隐私** | GDPR/CCPA、数据保护、用户权利响应、EDM合规 |
| **产品与知识产权** | IP排查、产品安全认证、广告真实性、HS编码 |
| **支付与资金** | PCI DSS、拒付率风控、资金链路透明化 |
| **税务与报关** | 国内税务申报、目的国税务、进出口报关规范 |
| **物流与售后** | 物流轨迹透明、售后与退货政策 |

## 7. 前后端交互规范

### 7.1 三种通信模式

| 模式 | 传输层 | 方向 | 适用场景 |
|------|-------|------|---------|
| **REST CRUD** | HTTP请求/响应 | 双向 | 配置管理(增删改查) |
| **SSE流式** | HTTP长连接 | 服务端推送 | 对话/AI生成 |
| **WebSocket** | WebSocket全双工 | 双向 | 实时通知/事件推送 |

### 7.2 SSE流式对话交互

#### 完整事件序列

```
thinking → plan → token(流式文本) → skill_start → token → skill_end → token → action_card → done
```

#### 事件类型定义

| event类型 | 触发时机 | data结构 | 前端处理 |
|-----------|---------|---------|---------|
| `thinking` | NLU意图解析 | `{content, depth}` | 显示思考过程 |
| `plan` | 多阶段规划 | `{steps, current_step}` | 显示规划面板 |
| `token` | 流式文本生成 | `{content}` | 追加到textContent |
| `skill_start` | Skill开始执行 | `{skill_name, args}` | 显示Skill调用卡片 |
| `skill_end` | Skill执行完成 | `{skill_name, result}` | 关闭Skill卡片 |
| `action_card` | 合规操作建议 | `{title, actions}` | 渲染操作按钮组 |
| `done` | 流式完成 | `{finish_reason, usage}` | 标记isStreaming=false |

### 7.3 WebSocket实时推送

#### 连接生命周期

```
App启动 → 用户登录 → WebSocket连接 → 心跳维持(30s) → 监听事件 → 断开自动重连(5s)
```

#### 已知事件类型

- `session_update` → 会话列表变更
- `new_message` → 新AI回复消息
- `alert` → 风险预警
- `scan_update` → 扫描进度更新
- `product_created` → 新产品创建
- `compliance_check_failed` → 合规检查失败
- `certification_expiry` → 认证到期预警
- `regulation_change` → 法规变更通知

### 7.4 认证交互流程

```
前端POST /api/v1/auth/login → 后端验证 → JWT创建(HS256) → 返回access_token → 
前端存储localStorage → 后续请求携带Bearer Token → 后端OAuth2PasswordBearer校验
```

### 7.5 错误处理体系

| 场景 | 错误类型 | 前端处理 | 用户可见反馈 |
|------|---------|---------|------------|
| 后端未启动 | 网络错误 | catch→降级/空状态 | 空白列表/错误提示 |
| Token过期 | HTTP 401 | AuthContext检测 | 重定向到登录页 |
| 权限不足 | HTTP 403 | try/catch→alert | 需要管理员权限 |
| 数据不存在 | HTTP 404 | try/catch→fallback | 默认值/空状态 |
| 服务端异常 | HTTP 500 | try/catch→error state | 加载失败提示 |
| SSE连接中断 | AbortError | 自动标记status='error' | 状态指示变红 |
| WebSocket断开 | onclose事件 | 5s后自动重连 | 短暂状态变化 |

## 8. 配置管理

### 8.1 三级配置架构

```
Level 1: 页面交互层 (React Page)
    ↓ 用户操作触发API调用
Level 2: API层 (config.ts)
    ↓ HTTP请求
Level 3: 后端持久化层 (SQLite/JSON/Markdown)
```

### 8.2 配置生效链路

| 配置项 | 前端写入 | 后端持久化 | 运行时消费 | 生效时机 |
|--------|---------|-----------|-----------|---------|
| **Agent** | agentsApi.create/update | SQLite | agent_config_store | 下次对话 |
