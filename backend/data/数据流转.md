## 二、API 数据流（前端 ↔ 后端）

### 2.1 用户认证

```
用户 → 登录页面 → POST /api/v1/auth/login → user_store 验证密码
    → 返回 JWT Token → localStorage 存储 → 后续请求 Bearer Token
```

| 文件 | 说明 |
|------|------|
| `frontend/src/api/config.ts` | 前端 API 客户端，统一 `authHeaders()` 附加 Token |
| `backend/app/api/auth.py` | 登录/注册/改密码端点 |
| `backend/app/core/auth.py` | JWT 生成/验证 |
| `backend/app/storage/user_store.py` | SQLite 用户存储 |

### 2.2 SSE 流式对话（核心入口）

```
POST /api/v1/chat/stream

前端: EventSource / fetch + ReadableStream → SSE 事件流
后端:
  用户消息 → RBAC检查 → NLU意图解析 → 分支路由
    ├─ 合规查询: RuleEngine → RAG → 合规报告 → SSE token事件
    └─ 通用问题: AstraAssistant SDK 对话 → SSE token事件

SSE 事件协议:
  event: thinking    → 思考过程
  event: plan       → 执行计划步骤
  event: token      → 文本令牌
  event: skill_start → Skill 开始执行
  event: skill_end  → Skill 执行完成
  event: action_card → 推荐操作卡片
  event: error      → 错误信息
  event: done       → 完成
```

| 文件 | 说明 |
|------|------|
| `frontend/src/api/config.ts` | 前端 API 客户端（无直接 SSE 封装，使用原生 EventSource） |
| `backend/app/api/chat_stream.py` | SSE 流式对话主入口（563行，原 streaming.py 拆分） |
| `backend/app/core/nlu.py` | NLU 意图解析 |
| `backend/app/services/astra_assistant.py` | Claude SDK 对话封装 |

### 2.3 Agent 管理

```
前端 Agent 配置页 → agentsApi → REST API

GET    /api/v1/agents           → 列表（含 system_prompt 预览）
GET    /api/v1/agents/{id}      → 详情
POST   /api/v1/agents           → 新建 (admin)
PUT    /api/v1/agents/{id}      → 更新 (admin)
DELETE /api/v1/agents/{id}      → 删除 (admin)
PUT    /api/v1/agents/{id}/toggle → 启用/禁用 (admin)

Agent 关联配置:
GET/PUT /api/v1/agents/{id}/skills → Skills 关联
GET/PUT /api/v1/agents/{id}/tools  → Tools 关联
GET/PUT /api/v1/agents/{id}/oauth  → OAuth 关联

Agent 运行时:
GET    /api/v1/agents/{id}/status → 状态（含关联 skill/tool/oauth）
POST   /api/v1/agents/{id}/chat  → Agent 身份对话
POST   /api/v1/agents/{id}/chat/stream → Agent 身份流式对话

Agent 任务:
GET  /api/v1/agents/tasks       → 活跃任务列表
GET  /api/v1/agents/workers     → Worker 状态
GET  /api/v1/agents/templates   → 任务分解模板

存储：
  agent_config 表 (SQLite) → agent_config_store.py
  data/agents/*.md          → agent_md_loader.py（文件驱动，启动时扫描）
  data/agents/extensions.json → Skill/Tools/OAuth 关联

消费链路:
  对话时 → AstraAssistant.run_as_agent(agent_id) →
  读取 SQLite agent_config → 构造 ClaudeAgentOptions → SDK 对话
```

| 文件 | 说明 |
|------|------|
| `backend/app/api/agent_crud.py` | Agent CRUD 增删改查 + 开关（207行，原 agent_config.py 拆分） |
| `backend/app/api/agent_extensions.py` | Agent-Skills/Tools/OAuth 关联 + 状态（130行） |
| `backend/app/api/agent_config.py` | Agent 身份对话执行（chat/chat/stream） |
| `backend/app/storage/agent_config_store.py` | SQLite 存储（387行） |
| `backend/app/storage/agent_md_loader.py` | Agent .md 文件加载器（新增，文件驱动） |

## 三、核心数据流场景

### 3.1 合规查询完整数据流

```
用户输入 "手机出口德国需要什么认证"
  │
  ├─▶ POST /api/v1/chat/stream (SSE)
  │     └─▶ chat_stream.py
  │
  ├─▶ Step 1: RBAC 权限检查 (如果启用)
  │
  ├─▶ Step 2: NLU 意图解析
  │     ├─ 读取 L3 user_memory (偏好)
  │     ├─ 读取 L4 session (上下文)
  │     └─ 返回: {product: "手机", country: "德国", action: "compliance"}
  │
  ├─▶ Step 3: 事件总线发布意图事件
  │     └─▶ EventBus.publish({type: "user:intent_parsed", ...})
  │
  ├─▶ Step 4: 规则引擎合规检查
  │     ├─ 读取 L0 hs_codes / vat_rates / cert_matrix
  │     ├─ 调用 RuleEngine.check_compliance("手机", "德国")
  │     └─ 返回 HS编码/VAT税率/认证要求
  │
  ├─▶ Step 5: Skill 推荐
  │     ├─ SkillRecommender.recommend_by_context({stage, category})
  │     └─ 返回关联 Skills
  │
  ├─▶ Step 6: RAG 法规检索
  │     ├─ 读取 L1 ChromaDB (语义搜索)
  │     └─ 返回法规片段
  │
  ├─▶ Step 7: 组装合规报告 (SSE token事件)
  │
  ├─▶ Step 8: 持久化
  │     ├─ 写入 L2 project_memory (产品合规档案)
  │     ├─ 写入 L4 session_memory (会话记录)
  │     └─ 写入 L5 event_chain (操作链)
  │
  └─▶ Step 9: 发布合规事件
        └─▶ EventBus.publish({type: "compliance:check_passed", ...})
```

### 3.2 多Agent任务执行流

```
POST /api/v1/agents/tasks { task: "检查所有产品CE认证", ... }
  │
  ├─▶ ManagerAgent.submit_task()
  │     ├─▶ TaskDecomposer.decompose() → 任务分解
  │     ├─▶ _find_best_worker()       → 匹配最佳Worker
  │     │     └─ WorkerRegistry 读取 workers/*.md
  │     └─▶ _run_worker() → 执行
  │           ├─ (首选) AstraAssistant SDK 对话
  │           └─ (降级) SkillExecutor 执行
  │
  ├─▶ 任务进度可轮询
  │     GET /api/v1/agents/tasks/{group_id}
  │
  └─▶ 用户可干预
        POST /api/v1/agents/tasks/{group_id}/intervene
```

### 3.3 QAAgent 系统自管理流

```
QAAgent 具有 22+ 内置管理工具:
  ┌─────────────────┬───────────────────┬──────────────────┐
  │ Worker管理       │ 事件管理          │ 规则/调度管理     │
  │ add_worker_type  │ add_event_type    │ add_rule         │
  │ modify_worker    │ modify_event      │ execute_cli      │
  │ delete_worker    │ delete_event      │ health_check     │
  └─────────────────┴───────────────────┴──────────────────┘

权限模型: safe (直接执行) / guarded (需确认) / blocked (拒绝)

调用方式:
  1. REST API 代理: worker_config/event_config 通过 QAAgent 执行
  2. CLI 命令: POST /api/v1/cli/execute → QAAgent.execute_cli_command
  3. 系统内部: 其他组件直接调用 QAAgent 方法
```

### 3.4 事件驱动流

```
事件定义 (data/events/builtin/*.md)
  │
  ├─▶ EventRegistry 启动时加载
  │
  └─▶ EventBus 运行时
        ├─ publish_raw() → 创建 EventRecord
        ├─ bus.on_all(manager.on_event)  ← ManagerAgent 全局监听
        │   └─▶ on_event() 4路分发:
        │       ├─▶ _route_to_websocket()      → ws_manager.send_to_user()
        │       │     session:created/deleted → session_update
        │       │     message:created         → new_message
        │       │     scan:*                  → scan_update
        │       │     risk:*/severity≥high   → alert
        │       ├─▶ _route_to_worker()          → 拆解子任务 + SDK 执行
        │       │     查找 EventDefinition.related_worker
        │       │     注入 tools/skills/agent_action 到 task_context
        │       ├─▶ _route_to_product_metrics()  → products/{id}/metrics/metrics.json
        │       │     event_count / last_event_type / event_type_counts
        │       └─▶ _route_to_custom_metrics()   → custom_metrics.json
        │             scope 匹配 → formula 求值 → 阈值告警
        ├─▶ NotificationEngine 通知引擎
        └─▶ Webhook (外部系统)
```

### 3.5 Scheduler 定时任务流

```
APScheduler (启动时 start_scheduler())
  │
  ├─ 全局任务 (GLOBAL_JOB_IDS):
  │   ├─ market_poll          → 市场扫描
  │   ├─ metrics_collect      → 指标收集
  │   ├─ proactive_daily_brief → 日报
  │   ├─ proactive_cert_expiry → 认证到期检查
  │   └─ proactive_regulation_scan → 法规变更扫描
  │
  ├─ 产品级任务 (按产品注册):
  │   ├─ check_cert_expiry_{product_id}
  │   └─ scan_regulation_changes_{product_id}
  │
  └─ 任务-Worker 绑定:
        Scheduler → Task → [绑定] → Worker (SDK执行)
        API: GET/PUT /api/v1/scheduler/bindings
```

## 四、关键架构注意事项

### 4.1 已知架构断裂点

| 问题 | 说明 | 涉及文件 |
|------|------|----------|
| **Tools 脚本化** | `_registry.yaml` 定义的 script 工具通过 astra_assistant.py MCP 工厂消费 | `tools.py`, `_registry.yaml`, `impl/`, `astra_assistant.py` |
| **模型配置双轨** | `model_router` 引擎与 `settings.sdk_model`（SDK对话）独立运行 | `model_config.py`, `model_router.py`, `config.py` |
| **Manager/QAAgent 不感知 Tools** | 两个核心 Agent 不含 tools.json 管理能力 | `manager_agent.py`, `qa_agent.py` |
| **Manager/QAAgent 不感知 ModelRouter** | 核心 Agent 不引用 ModelRouter | `manager_agent.py`, `qa_agent.py` |
