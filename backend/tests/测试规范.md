## 一、测试框架

| 组件 | 说明 |
|------|------|
| **框架** | pytest 8.x |
| **异步支持** | pytest-asyncio（auto 模式） |
| **HTTP 客户端** | httpx（AsyncClient + ASGITransport 直连） |
| **覆盖率** | pytest-cov |

### 配置文件

```ini
# pytest.ini
[pytest]
asyncio_mode = auto
testpaths = tests
markers =
    live: 需要运行中的后端服务器(localhost:8002)的全流程测试
    slow: 执行时间较长的测试
```

## 三、测试分层

### 3.1 单元测试（Unit）

- **文件**: `test_rule_engine.py`
- **范围**: 单一函数/类的逻辑验证
- **原则**: 无 I/O 依赖，无外部服务调用，毫秒级执行
- **覆盖**: HS编码查询、VAT计算、认证矩阵匹配、风险标志

### 3.2 接口契约测试（Contract）

- **文件**: `test_openapi_contract.py`
- **范围**: OpenAPI schema 完整性验证 + 各模块端点可达性
- **原则**: 验证端点注册、参数校验(422)、响应结构
- **特性**:
  - 维护 `EXPECTED_ENDPOINTS` 列表（160+端点契约）
  - 最低端点数门槛（≥15０）
  - 各模块独立端点功能验证

### 3.3 集成测试（Integration）

- **文件**: `test_chains.py`
- **范围**: 多模块协作、CRUD 完整性验证
- **原则**: 通过 ASGI 直连测试完整请求链路
- **覆盖**: ActionChain/EventChain 状态流转、NLStore CRUD

### 3.4 全业务流集成测试（Business Flow）

- **文件**: `test_full_business_flow.py`
- **范围**: 16个Phase覆盖完整业务场景
- **原则**: 真实HTTP调用，TestReport质量收集器
- **Phase列表**:
  1. 基础健康 & 认证
  2. Shopify OAuth & 店铺同步
  3. 产品管理
  4. Agent配置
  5. Skills管理
  6. Tools管理
  7. OAuth集成
  8. 合规流水线
  9. CLI命令
  10. SSE流式对话
  11. 风险预警 & 通知
  12. 指标监控 & 主动引擎
  13. 记忆 & 知识库
  14. 定时任务
  15. 权限管理
  16. 端到端场景

### 3.5 端到端测试（E2E）

- **文件**: `test_lifecycle_e2e.py`
- **范围**: 产品全生命周期 × 第三方系统集成
- **原则**: 模拟25+第三方系统Provider注册、通知频道、状态机流转
- **输出**: 中间结果写入系统临时目录（不污染项目）
- **质量评分**: 状态码30分 + 结构40分 + 业务逻辑30分

### 3.6 全流程系统测试（Live System） `@pytest.mark.live`

> 这类测试需要后端服务器实际运行，使用 httpx 同步客户端访问 `localhost:8002`。
> 运行方式: `pytest -m live`（需先启动后端服务）

| 文件 | 范围 | 特点 |
|------|------|------|
| `test_phase1.py` | Phase 1 专项测试 | 产品CRUD、事件总线、RAG、流水线、CLI、事件配置、Worker配置 |
| `test_all_phases.py` | Phase 1-4 全阶段综合 | 产品管理、事件总线、记忆树、指标、Tools、SSE对话、Agent调度、RBAC、审批、报表 |
| `test_comprehensive_flow.py` | 核心模块直调 | 直接调用后端核心模块，模拟虚拟数据验证内部逻辑 |

## 四、编写规范

### 4.1 fixture 使用

```python
# conftest.py 中定义公共 fixture
@pytest_asyncio.fixture
async def client():
    """创建异步HTTP测试客户端（ASGI直连，无需启动服务器）"""
    transport = ASGITransport(app=app)
    async with AsyncClient(transport=transport, base_url="http://test") as c:
        yield c
```

### 4.2 测试函数命名

```python
# 格式: test_<功能>_<场景>
async def test_health_check(client):
    """健康检查应返回 200 和服务信息"""
    ...

async def test_auth_invalid_credentials(client):
    """无效凭据应返回 401"""
    ...
```

### 4.3 断言规范

- 优先使用 `assert` 原生断言
- 状态码断言放在最前面
- 结构断言使用 `in` 或字段检查
- 禁止使用 `try/except` 吞掉测试异常

```python
resp = await client.get("/api/v1/health")
assert resp.status_code == 200
data = resp.json()
assert data["status"] == "ok"
assert "version" in data
```

### 4.4 新增端点时的测试要求

1. 在 `test_openapi_contract.py` 的 `EXPECTED_ENDPOINTS` 中添加条目
2. 为新模块添加至少一个端点可达性测试函数
3. 如涉及 CRUD，在 `test_chains.py` 或新建对应集成测试文件

## 五、运行方式

```bash
# 进入后端目录
cd backend

# 运行全部测试
pytest

# 运行单个文件
pytest tests/test_rule_engine.py

# 运行带覆盖率
pytest --cov=app --cov-report=html

# 运行指定标记
pytest -m "not slow"

# 详细输出
pytest -v --tb=short

# 仅运行快速测试（单元+契约）
pytest tests/test_rule_engine.py tests/test_openapi_contract.py tests/test_chains.py
```

## 六、测试数据管理原则

| 原则 | 说明 |
|------|------|
| **自包含** | 测试数据在 fixture/setup 中创建，teardown 中清理 |
| **不依赖外部状态** | 禁止依赖 sessions.db 或 session_memory 中的遗留数据 |
| **不污染项目目录** | 中间输出写入系统临时目录（`tempfile.gettempdir()`） |
| **幂等性** | 测试可重复执行，不因顺序或并发产生副作用 |
| **最小化** | 只创建当前测试所需的最小数据集 |
| **不写文件到 tests/** | 禁止测试代码向 tests/ 目录写入 .json/.log 等产物 |

## 七、CI/CD 集成指引

### 7.1 推荐流水线配置

```yaml
test:
  stage: test
  script:
    - cd backend
    - pip install -r requirements.txt
    - pytest --tb=short --junitxml=report.xml
  artifacts:
    reports:
      junit: backend/report.xml
```

### 7.2 质量门禁

| 指标 | 阈值 |
|------|------|
| 测试通过率 | 100% |
| 行覆盖率 | >= 60%（核心模块 >= 80%） |
| 新增代码覆盖 | >= 80% |
| OpenAPI端点注册 | >= 150 个 |

### 7.3 Live 测试与 CI 策略

带 `@pytest.mark.live` 标记的测试需要运行中的后端服务，CI 中可按需排除：

```bash
# CI默认：跑不依赖外部服务的测试
pytest -m "not live" --tb=short

# 全量测试（含 live，需先启动后端）
pytest --tb=short

# 仅跑 live 全流程测试
pytest -m live
```

## 八、注意事项

1. **不提交测试产物**: `.log`、`.json`（报告类）、`output/` 目录已在 `.gitignore` 中排除
2. **不硬编码路径**: 使用 `Path(__file__).parent` 或 `tempfile` 模块
3. **不依赖网络**: 使用 ASGI 直连或 mock，确保离线可运行
4. **异步测试标注**: 使用 `@pytest.mark.asyncio` 或配置 `asyncio_mode = auto`
5. **测试独立性**: 每个测试函数独立执行，不依赖其他测试的执行顺序
6. **端点契约同步**: 新增/删除 API 端点时同步更新 `EXPECTED_ENDPOINTS` 列表
