## 一、调研核心结论

### 1.1 最严重缺口（阻断生产使用）

| 缺口 | 现状 | 影响 | 优先级 |
|------|------|------|--------|
| **三单一致性** | Prompt 声明，无实现；订单表根本不存在 | 报关成功率 -30%+ | P0 |
| **报关单字段** | 缺品牌/规格/发货人/收货人/合同号 | 海关退单，无法申报 | P0 |
| **Webhook 安全** | 无签名验证，任意伪造；异常全吞掉无日志 | 数据被篡改，问题不可追踪 | P0 |
| **管制品检测** | 无数据库，仅靠 LLM 判断 | 法律风险（OFAC/新疆棉） | P0 |
| **关税税率** | 7 国 16 品类，覆盖率 ~1.3% | 大量品类无法计算 | P1 |
| **17TRACK 追踪** | API Key 未配置，实际未接入 | 轨迹更新全失效 | P1 |

### 1.2 技术验证结论

```
17TRACK API  → 需配置 TRACK17_API_KEY；接口格式 ✅，401 认证可通过
WTO API      → 需配置 WTO_API_KEY；或改用离线 JSON 数据包（推荐）
三单一致性   → DB 无订单表，需先建 sales_orders + payment_records 表
Webhook 安全 → HMAC-SHA256 验证，17TRACK 用 secret token，AfterShip 用 hmac-sha256
```

## 二、三单一致性 — 核心设计

### 2.1 三单定义

```
销售订单（销售单）   →  platform_order_id / buyer / items / amount
支付单（资金流）     →  payment_ref / amount / currency / payer
物流单/报关单        →  tracking_number / declared_value / declared_name
```

### 2.2 需要新建的数据表

```sql
-- 销售订单（来源：Shopify/手动录入）
```

## 四、关税税率数据方案

### 4.1 策略选择

| 方案 | 覆盖率 | 成本 | 实时性 | 推荐 |
|------|--------|------|--------|------|

| 米塔搜索降级 | 模糊 | API 消耗 | 实时 | 🟡 兜底 |
| 商业税率库（Import Genius 等） | 全量 | 付费 | 实时 | 🔵 未来 |

### 4.2 优先扩充的国家和 HS 编码

**新增国家（按跨境电商体量排序）**：
```
第一批：SG(新加坡) MY(马来西亚) TH(泰国) VN(越南) IN(印度) BR(巴西)
第二批：MX(墨西哥) AE(阿联酋) SA(沙特) ZA(南非) TR(土耳其) PL(波兰)
```

**新增 HS 编码品类（补充电商高频品类）**：
```
玩具/游戏    95xx  → 美国 0-19%，差异极大
食品        02-22  → 多国敏感品类
化妆品      3304   → 欧盟 0%，美国 0%，但需 FDA
珠宝        7113   → 多国 0-5%
```

### 4.3 税率查询降级链

```
精确匹配（8位HS + 目的国）
    ↓ 失败
前缀匹配（4位→2位，使用子品类均值）
    ↓ 失败
_default（国家平均税率）
    ↓ 失败
米塔搜索（"HS {code} {country} tariff rate {year}"）
    ↓ 失败
返回 0%（标注"税率未知，建议人工核查"）
```

```python
import hmac, hashlib

def verify_17track_signature(payload_bytes: bytes, signature: str, secret: str) -> bool:
    """HMAC-SHA256 签名验证。"""
    expected = hmac.new(secret.encode(), payload_bytes, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, signature.lower())
```

### 7.1 完整的安全处理流程

```python
# 强化后的 Webhook 处理架构：

@router.post("/webhook/17track")
async def webhook_17track(request: Request):
    # Step 1: 签名验证
    raw_body = await request.body()
    signature = request.headers.get("X-17Track-Signature", "")
    if not _verify_17track_sig(raw_body, signature):
        raise HTTPException(401, "签名验证失败")
    
    # Step 2: 解析 payload
    try:
        payload = json.loads(raw_body)
    except json.JSONDecodeError:
        raise HTTPException(400, "无效 JSON")
    
    # Step 3: 幂等性检查（防重复处理）
    event_id = request.headers.get("X-17Track-Event-Id", "")
    if event_id and _is_processed(event_id):
        return {"received": True, "skipped": True}  # 幂等返回
    
    # Step 4: 异步处理（快速响应，防超时）
    background_tasks.add_task(_process_17track_payload, payload)
    
    # Step 5: 立即返回 200（Webhook 超时会重试）
    return {"received": True}
```
