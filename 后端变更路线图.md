## 二、核心能力矩阵（保留原有 + 业务流程实现）

### 2.1 OS级智能体8大能力（保留）

| 能力 | 说明 | 参考实现 | 业务流程应用 |
|------|------|----------|--------------|
| **事件驱动合规流** | 感知→检查→推荐→告知→交互→处理 六阶段闭环（与指南§6.15对齐；内部5阶段可将推荐与告知视为一个逻辑阶段） | OpenHuman 事件驱动 | 贯穿10大业务阶段的核心流程 |
| **多Agent调度** | Manager协调 + Worker隔离 + 群聊式调度 | QwenPaw 多智能体 | 不同业务阶段由不同Worker Agent处理 |
| **记忆树** | 层级化摘要树 + SQLite存储 + Obsidian Wiki输出 | OpenHuman 记忆树 | 产品级记忆存储，支持全生命周期追溯 |
| **TokenJuice** | 智能Token压缩（HTML→MD、URL缩短、冗余去除） | OpenHuman TokenJuice | 压缩Shopify API响应，优化上下文窗口 |
| **QAAgent** | 系统自我管理（配置问答/修改、事件定义、流程串联） | 自研 | 管理业务规则配置，定义新事件类型 |
| **第三方集成** | OAuth接入 + 每20分钟自动拉取新数据 | OpenHuman 集成层 | Shopify OAuth接入，自动拉取产品/订单数据 |
| **主动引擎** | 定时任务、心跳自检、洞察挖掘 | QwenPaw 心跳 | 认证到期预警、法规变更扫描、合规简报 |
| **安全沙箱** | 工具防护、文件防护、技能安全扫描 | QwenPaw 安全特性 | Skills执行安全验证，敏感操作审批 |

```python
# backend/app/core/event_driven_compliance.py

class EventDrivenComplianceFlow:
    """事件驱动合规流（六阶段闭环）"""
    
    def __init__(
        self,
        event_chain: EventChain,           # 原有模块
        rule_engine: RuleEngine,           # 原有模块
        market_monitor: MarketMonitor,     # 原有模块
        notification_engine: NotificationEngine,
        astra_assistant: AstraAssistant,   # 原有模块
        action_chain: ActionChain
    ):
        self.event_chain = event_chain
        self.rule_engine = rule_engine
        self.market_monitor = market_monitor
        self.notification = notification_engine
        self.assistant = astra_assistant
        self.action_chain = action_chain
    
    async def execute(self, trigger_event: EventRecord) -> ComplianceResult:
        """执行六阶段闭环"""
        
        # 阶段1：感知
        # EventChain接收事件，MarketMonitor补充市场数据
        enriched_event = await self._perceive(trigger_event)
        
        # 阶段2：检查
        # RuleEngine执行规则检查，RAG检索相关法规
        check_result = await self._check(enriched_event)
        
        # 阶段3：告知
        # 通知引擎推送结果到多渠道
        await self._notify(enriched_event, check_result)
        
        # 阶段4：交互
        # AstraAssistant提供对话式交互，Skills执行操作
        user_action = await self._interact(enriched_event, check_result)
        
        # 阶段5：处理
        # ActionChain执行用户确认的操作
        result = await self._process(user_action)
        
        # 结果回写到EventChain
        await self.event_chain.append(result.to_event())
        
        return result
    
    async def _perceive(self, event: EventRecord) -> EnrichedEvent:
        """阶段1：感知"""
        # 原有EventChain存储事件
        await self.event_chain.append(event)
        
        # 原有MarketMonitor补充市场数据
        market_data = await self.market_monitor.get_market_context(event.product_id)
        
        return EnrichedEvent(
            original=event,
            market_data=market_data
        )
    
    async def _check(self, event: EnrichedEvent) -> CheckResult:
        """阶段2：检查"""
        # 原有RuleEngine执行规则检查
        rule_result = await self.rule_engine.evaluate(event)
        
        # 原有RAG检索相关法规
        regulations = await self.rag.search(event.product_id, event.type)
        
        return CheckResult(
            rule_result=rule_result,
            regulations=regulations,
            risk_level=self._calculate_risk_level(rule_result)
        )
    
    async def _notify(self, event: EnrichedEvent, check_result: CheckResult):
        """阶段3：告知"""
        # 通知引擎推送到多渠道
        await self.notification.send(
            event_type=event.type,
            severity=check_result.risk_level,
            channels=["dashboard", "websocket"],
            content={
                "event": event.to_dict(),
                "check_result": check_result.to_dict()
            }
        )
    
    async def _interact(self, event: EnrichedEvent, check_result: CheckResult) -> UserAction:
        """阶段4：交互"""
        # 原有AstraAssistant提供对话式交互
        recommendations = await self.assistant.generate_recommendations(
            event=event,
            check_result=check_result
        )
        
        # 等待用户确认
        user_action = await self.assistant.wait_for_user_action(recommendations)
        
        return user_action
    
    async def _process(self, user_action: UserAction) -> ProcessResult:
        """阶段5：处理"""
        # ActionChain执行操作
        result = await self.action_chain.execute(user_action)
        
        return result
```

```python
# backend/app/core/event_registry.py

import yaml
from pathlib import Path
from typing import Dict, List, Optional

class EventRegistry:
    """事件注册表 - 配置文件驱动"""
    
    def __init__(self, config_dir: str = "data/config/events"):
        self.config_dir = Path(config_dir)
        self._events: Dict[str, EventDefinition] = {}
        self._load_all_events()
    
    def _load_all_events(self):
        """加载所有事件配置"""
        for md_file in self.config_dir.glob("*.md"):
            if md_file.name.startswith("_"):
                continue
            self._load_events_from_file(md_file)
    
    def _load_events_from_file(self, file_path: Path):
        """从Markdown文件加载事件定义"""
        content = file_path.read_text(encoding="utf-8")
        # 解析Markdown表格中的事件定义
        events = self._parse_event_table(content)
        
        for event_def in events:
            self._events[event_def.event_code] = event_def
    
    def _parse_event_table(self, content: str) -> List[EventDefinition]:
        """解析Markdown表格"""
        # 解析 | 事件编码 | 事件名称 | ... | 格式的表格
        events = []
        lines = content.split("\n")
        in_table = False
        headers = []
        
        for line in lines:
            if "|" in line:
                cells = [c.strip() for c in line.split("|") if c.strip()]
                
                if not in_table:
                    headers = cells
                    in_table = True
                elif "---" not in cells[0]:  # 跳过分隔行
                    event = self._create_event_from_row(headers, cells)
                    if event:
                        events.append(event)
            else:
                in_table = False
                headers = []
        
        return events
    
    def _create_event_from_row(self, headers: List[str], cells: List[str]) -> Optional[EventDefinition]:
        """从表格行创建事件定义"""
        if len(cells) < 4:
            return None
        
        data = dict(zip(headers, cells))
        
        return EventDefinition(
            event_code=data.get("事件编码", ""),
            event_name=data.get("事件名称", ""),
            business_stage=data.get("业务阶段", ""),
            trigger_condition=data.get("触发条件", ""),
            related_worker=data.get("关联Worker", ""),
            severity=data.get("严重级别", "low"),
            notify_strategy=data.get("通知策略", "dashboard").split(",")
        )
    
    def get_event(self, event_code: str) -> Optional[EventDefinition]:
        """获取事件定义"""
        return self._events.get(event_code)
    
    def get_all_events(self) -> List[EventDefinition]:
        """获取所有事件定义"""
        return list(self._events.values())
    
    def get_events_by_stage(self, stage: str) -> List[EventDefinition]:
        """按业务阶段获取事件"""
        return [e for e in self._events.values() if e.business_stage == stage]
    
    async def register_event(self, event_def: EventDefinition, file_name: str = "custom_events.md"):
        """注册新事件（写入配置文件）"""
        file_path = self.config_dir / file_name
        
        # 读取现有内容
        content = file_path.read_text(encoding="utf-8") if file_path.exists() else "# 自定义事件定义\n\n| 事件编码 | 事件名称 | 业务阶段 | 触发条件 | 关联Worker | 严重级别 | 通知策略 |\n|----------|----------|----------|----------|------------|----------|----------|\n"
        
        # 追加新事件行
        new_row = f"| {event_def.event_code} | {event_def.event_name} | {event_def.business_stage} | {event_def.trigger_condition} | {event_def.related_worker} | {event_def.severity} | {','.join(event_def.notify_strategy)} |\n"
        content += new_row
        
        # 写入文件
        file_path.write_text(content, encoding="utf-8")
        
        # 更新内存缓存
        self._events[event_def.event_code] = event_def
        
        return True
    
    async def delete_event(self, event_code: str):
        """删除事件（归档到_archive目录）"""
        event = self._events.get(event_code)
        if not event:
            return False
        
        # 从所有配置文件中移除该事件
        for md_file in self.config_dir.glob("*.md"):
            if md_file.name.startswith("_"):
                continue
            await self._remove_event_from_file(md_file, event_code)
        
        # 归档到_archive目录
        archive_dir = self.config_dir / "_archive"
        archive_dir.mkdir(exist_ok=True)
        archive_file = archive_dir / f"{event_code.replace(':', '_')}.md"
        archive_file.write_text(f"# 已删除事件: {event_code}\n\n删除时间: {datetime.utcnow().isoformat()}\n\n原始定义:\n{event.to_dict()}", encoding="utf-8")
        
        # 从内存缓存移除
        del self._events[event_code]
        
        return True
    
    async def _remove_event_from_file(self, file_path: Path, event_code: str):
        """从文件中移除事件行"""
        content = file_path.read_text(encoding="utf-8")
        lines = content.split("\n")
        new_lines = []
        
        for line in lines:
            if event_code not in line:
                new_lines.append(line)
        
        file_path.write_text("\n".join(new_lines), encoding="utf-8")

```python
# backend/app/core/qa_agent.py (事件管理部分)

class QAAgent:
    """系统自我管理智能体"""
    
    def __init__(self, event_registry: EventRegistry, worker_registry: WorkerRegistry):
        self.event_registry = event_registry
        self.worker_registry = worker_registry
    
    async def add_event_type(
        self,
        event_code: str,
        event_name: str,
        business_stage: str,
        trigger_condition: str,
        related_worker: str = "",
        severity: str = "low",
        notify_strategy: List[str] = ["dashboard"]
    ) -> bool:
        """添加新事件类型"""
        # 验证事件编码唯一性
        if self.event_registry.get_event(event_code):
            raise ValueError(f"事件编码 {event_code} 已存在")
        
        # 验证关联Worker是否存在
        if related_worker and not self.worker_registry.get_worker(related_worker):
            raise ValueError(f"Worker {related_worker} 不存在")
        
        # 创建事件定义
        event_def = EventDefinition(
            event_code=event_code,
            event_name=event_name,
            business_stage=business_stage,
            trigger_condition=trigger_condition,
            related_worker=related_worker,
            severity=severity,
            notify_strategy=notify_strategy
        )
        
        # 注册到配置文件
        await self.event_registry.register_event(event_def)
        
        return True
    
    async def modify_event_type(
        self,
        event_code: str,
        **kwargs
    ) -> bool:
        """修改事件类型"""
        event = self.event_registry.get_event(event_code)
        if not event:
            raise ValueError(f"事件 {event_code} 不存在")
        
        # 更新字段
        for key, value in kwargs.items():
            if hasattr(event, key):
                setattr(event, key, value)
        
        # 重新写入配置文件
        await self.event_registry.update_event(event)
        
        return True
    
    async def delete_event_type(self, event_code: str) -> bool:
        """删除事件类型"""
        return await self.event_registry.delete_event(event_code)
    
    async def list_event_types(self, stage: str = None) -> List[EventDefinition]:
        """列出事件类型"""
        if stage:
            return self.event_registry.get_events_by_stage(stage)
        return self.event_registry.get_all_events()

# backend/app/core/worker_registry.py

import yaml
from pathlib import Path
from typing import Dict, List, Optional

class WorkerRegistry:
    """Worker注册表 - 配置文件驱动"""
    
    def __init__(self, config_dir: str = "data/config/workers"):
        self.config_dir = Path(config_dir)
        self._workers: Dict[str, WorkerDefinition] = {}
        self._load_all_workers()
    
    def _load_all_workers(self):
        """加载所有Worker配置"""
        for md_file in self.config_dir.glob("*.md"):
            if md_file.name.startswith("_"):
                continue
            self._load_workers_from_file(md_file)
    
    def _load_workers_from_file(self, file_path: Path):
        """从Markdown文件加载Worker定义"""
        content = file_path.read_text(encoding="utf-8")
        workers = self._parse_worker_table(content)
        
        for worker_def in workers:
            self._workers[worker_def.worker_code] = worker_def
    
    def _parse_worker_table(self, content: str) -> List[WorkerDefinition]:
        """解析Markdown表格"""
        workers = []
        lines = content.split("\n")
        in_table = False
        headers = []
        
        for line in lines:
            if "|" in line:
                cells = [c.strip() for c in line.split("|") if c.strip()]
                
                if not in_table:
                    headers = cells
                    in_table = True
                elif "---" not in cells[0]:
                    worker = self._create_worker_from_row(headers, cells)
                    if worker:
                        workers.append(worker)
            else:
                in_table = False
                headers = []
        
        return workers
    
    def _create_worker_from_row(self, headers: List[str], cells: List[str]) -> Optional[WorkerDefinition]:
        """从表格行创建Worker定义"""
        if len(cells) < 4:
            return None
        
        data = dict(zip(headers, cells))
        
        return WorkerDefinition(
            worker_code=data.get("Worker编码", ""),
            worker_name=data.get("Worker名称", ""),
            business_stage=data.get("业务阶段", ""),
            description=data.get("职责描述", ""),
            available_skills=data.get("可用Skills", "").split(","),
            priority=int(data.get("优先级", "5")),
            resource_limit=data.get("资源限制", "")
        )
    
    def get_worker(self, worker_code: str) -> Optional[WorkerDefinition]:
        """获取Worker定义"""
        return self._workers.get(worker_code)
    
    def get_all_workers(self) -> List[WorkerDefinition]:
        """获取所有Worker定义"""
        return list(self._workers.values())
    
    def get_workers_by_stage(self, stage: str) -> List[WorkerDefinition]:
        """按业务阶段获取Worker"""
        return [w for w in self._workers.values() if w.business_stage == stage]
    
    async def register_worker(self, worker_def: WorkerDefinition, file_name: str = "custom_workers.md"):
        """注册新Worker（写入配置文件）"""
        file_path = self.config_dir / file_name
        
        # 读取现有内容
        content = file_path.read_text(encoding="utf-8") if file_path.exists() else "# 自定义Worker定义\n\n| Worker编码 | Worker名称 | 业务阶段 | 职责描述 | 可用Skills | 优先级 | 资源限制 |\n|------------|------------|----------|----------|------------|--------|----------|\n"
        
        # 追加新Worker行
        new_row = f"| {worker_def.worker_code} | {worker_def.worker_name} | {worker_def.business_stage} | {worker_def.description} | {','.join(worker_def.available_skills)} | {worker_def.priority} | {worker_def.resource_limit} |\n"
        content += new_row
        
        # 写入文件
        file_path.write_text(content, encoding="utf-8")
        
        # 更新内存缓存
        self._workers[worker_def.worker_code] = worker_def
        
        return True
    
    async def delete_worker(self, worker_code: str):
        """删除Worker（归档到_archive目录）"""
        worker = self._workers.get(worker_code)
        if not worker:
            return False
        
        # 从所有配置文件中移除
        for md_file in self.config_dir.glob("*.md"):
            if md_file.name.startswith("_"):
                continue
            await self._remove_worker_from_file(md_file, worker_code)
        
        # 归档
        archive_dir = self.config_dir / "_archive"
        archive_dir.mkdir(exist_ok=True)
        archive_file = archive_dir / f"{worker_code}.md"
        archive_file.write_text(f"# 已删除Worker: {worker_code}\n\n删除时间: {datetime.utcnow().isoformat()}\n\n原始定义:\n{worker.to_dict()}", encoding="utf-8")
        
        # 从内存缓存移除
        del self._workers[worker_code]
        
        return True
```

    id TEXT PRIMARY KEY,            -- UUID

    title TEXT NOT NULL,            -- 摘要标题

    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    child_count INTEGER DEFAULT 0,   -- 子节点数量
    FOREIGN KEY (parent_id) REFERENCES summaries(id)
);

-- 索引

CREATE INDEX idx_fragments_parent ON fragments(parent_id);

class AutoPullEngine:
    """自动拉取引擎 - 每20分钟遍历所有活跃连接"""
    
    interval = 20 * 60  # 20分钟
    
    def __init__(self, oauth_manager: OAuthManager, memory_tree: MemoryTree):
        self.oauth = oauth_manager
        self.memory = memory_tree
        self.last_sync: Dict[str, datetime] = {}
    
    async def pull_cycle(self):
        """执行一次拉取周期"""
        connections = await self.oauth.get_active_connections()
        
        for conn in connections:
            try:
                new_data = await conn.fetch_incremental(
                    last_sync=self.last_sync.get(conn.id)
                )
                
                if new_data:
                    # TokenJuice压缩
                    compressed = await self.token_juice.compress(new_data)
                    
                    # 写入记忆树
                    await self.memory.ingest(compressed, source=conn.id)
                    
                    # 更新同步时间
                    self.last_sync[conn.id] = datetime.utcnow()
                    
                    # 写入事件
                    await self.event_bus.publish(EventRecord(
                        type="sync:completed",
                        source=conn.provider,
                        data={"records": len(new_data)}
                    ))
            
            except Exception as e:
                # 记录错误
                await self.event_bus.publish(EventRecord(
                    type="sync:failed",
                    source=conn.provider,
                    error=str(e)
                ))
```

# backend/app/core/security_sandbox.py

class SecuritySandbox:
    """安全沙箱"""
    
    # 危险命令黑名单
    BLOCKED_COMMANDS = [
        "rm -rf", "DROP TABLE", "DELETE FROM",
        "fork bomb", ":(){ :|:& };:"
    ]
    
    # 允许的文件路径
    ALLOWED_PATHS = ["data/**", "config/**"]
    
    # 禁止的文件路径
    FORBIDDEN_PATHS = ["~/.ssh/**", "~/.env", "/etc/**"]
    
    # 敏感文件模式
    SENSITIVE_PATTERNS = ["*.key", "*.pem", ".env*"]
    
    async def validate_command(self, command: str) -> GuardResult:
        """验证命令安全性"""
        for blocked in self.BLOCKED_COMMANDS:
            if blocked in command:
                return GuardResult(
                    allowed=False,
                    reason=f"危险命令: {blocked}"
                )
        return GuardResult(allowed=True)
    
    async def validate_path(self, path: str, operation: str) -> GuardResult:
        """验证路径访问权限"""
        # 检查禁止路径
        for forbidden in self.FORBIDDEN_PATHS:
            if self._match_path(path, forbidden):
                return GuardResult(
                    allowed=False,
                    reason=f"禁止访问: {forbidden}"
                )
        
        # 检查敏感文件
        for pattern in self.SENSITIVE_PATTERNS:
            if self._match_pattern(path, pattern):
                return GuardResult(
                    allowed=False,
                    reason=f"敏感文件需要确认: {path}",
                    need_confirm=True
                )
        
        return GuardResult(allowed=True)
    
    async def scan_skill(self, skill_path: str) -> ScanResult:
        """扫描技能安全性"""
        checks = [
            self._check_prompt_injection(skill_path),
            self._check_command_injection(skill_path),
            self._check_hardcoded_secrets(skill_path),
            self._check_data_exfiltration_risk(skill_path),
        ]
        
        results = await asyncio.gather(*checks)
        
        return ScanResult(
            passed=all(r.passed for r in results),
            issues=[r for r in results if not r.passed]
        )

#### 6.9.3 SSE流式对话与对话配置API（新增）

> SSE流式对话为前端最高优先级能力。参照前端SSE消息协议完整实现。

**SSE消息协议标准（对齐前端协议）：**

> **SSE传输层格式**（标准Server-Sent Events协议）：
> ```
> event: token|skill_start|skill_end|thinking|plan|action_card|error|done
> data: { ... }
> ```
> 前端通过 `EventSource.addEventListener('token', callback)` 监听各事件类型。
> 后端接口文档中TypeScript interface的 `type` 字段名对应SSE wire格式的 `event` 字段。

```typescript
// SSE事件类型（对齐前端协议：顶层字段名为 type 对应SSE event字段）
interface SSEEvent<T = any> {
  type: 'token' | 'skill_start' | 'skill_end' | 'thinking' | 'plan' | 'action_card' | 'error' | 'done';
  data: T;
}

// 各事件data载荷类型（★标注前端必须消费的字段，其余为扩展信息，前端可忽略；扩展字段全为可选）
type SSEPayloads = {
  token:        { content: string };                           // ★ content 文本内容
  skill_start:  { skill: string; args: Record<string, any> };  // ★ skill/args
  skill_end:    { skill: string; result: any; duration_ms?: number; status: 'success' | 'error' };  // ★ status 必消费, duration_ms 可选扩展
  thinking:     { content: string; depth?: number };             // ★ content 必消费, depth 可选扩展
  plan:         { steps: Array<{ id: string; action: string; expected_result?: string; status: 'pending' | 'running' | 'done' | 'failed'; skill?: string; duration_ms?: number }>; current: number };  // ★ steps/current, 对齐前端PlanStep
  action_card:  { actions: Array<{ id: string; label: string; description?: string; skill?: string; confidence?: number; expected_result?: string; risk_level?: 'low' | 'medium' | 'high'; status?: 'pending' | 'confirmed' | 'executing' | 'done' | 'skipped' }> };  // ★ actions, 对齐前端结构化Action
  error:        { code: string; message: string; recoverable?: boolean };  // ★ code/message, recoverable 可选
  done:         { finish_reason?: string; usage?: { prompt_tokens: number; completion_tokens: number } };  // finish_reason 可选扩展, usage 可选
};
```

```python
# backend/app/core/channel_adapter.py
class ChannelAdapter(ABC):
    """频道适配器抽象基类"""
    
    @abstractmethod
    async def send_message(self, target: str, content: str, 
                          attachments: list = None) -> str:
        """发送消息到目标频道"""
        pass
    
    @abstractmethod
    async def send_card(self, target: str, card: dict) -> str:
        """发送交互式卡片消息"""
        pass
    
    @abstractmethod
    async def send_notification(self, target: str, notification: dict) -> str:
        """发送通知（含 product_id 深度链接便于前端跳转）"""
        pass
    
    async def broadcast(self, targets: list[str], content: str):
        """向多个频道广播"""
        for t in targets:
            await self.send_message(t, content)

class FeishuAdapter(ChannelAdapter):
    """飞书机器人频道适配器"""
    pass

    """钉钉机器人频道适配器"""

# backend/app/models/schemas.py
from pydantic import BaseModel, Field
from typing import Optional, List, Dict, Any
from datetime import datetime
from enum import Enum

class EventCategory(str, Enum):
    lifecycle = "lifecycle"
    compliance = "compliance"
    certification = "certification"
    order = "order"
    regulation = "regulation"
    risk_alert = "risk_alert"
    system = "system"
    user_action = "user_action"

class DataSourceInfo(BaseModel):
    """数据血缘信息（存储层级映射表）"""
    read: List[str] = Field(default_factory=list, description="读取的数据源，如 ['L0:hs_codes', 'L2:product_meta']")
    write: List[str] = Field(default_factory=list, description="写入的数据源，如 ['L2:product_memory', 'L5:event_chain']")

class EventRecord(BaseModel):
    """标准化事件记录"""
    id: str = Field(default_factory=lambda: uuid.uuid4().hex[:12], description="事件唯一ID")
    type: str                                          # 事件类型（如 "product:created"）
    category: EventCategory = Field(default=EventCategory.system)  # 事件分类（8类）
    source: str                                        # 事件来源（如 "shopify:webhook", "scheduler", "user"）
    product_id: Optional[str] = None                   # 关联产品ID

    data: Dict[str, Any] = Field(default_factory=dict) # 事件载荷
    data_sources: DataSourceInfo = Field(default_factory=DataSourceInfo)  # ★ 数据血缘
    severity: str = "low"                              # 严重级别
    error: Optional[str] = None                        # 错误信息（失败时）
    created_at: str = Field(default_factory=lambda: datetime.utcnow().isoformat())

## 十、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| **原有功能兼容性** | 升级可能破坏现有功能 | 1. 渐进式升级 2. 充分测试 3. 回滚方案 |
| **产品级存储性能** | 大量产品时读写缓慢 | 1. 索引优化 2. 缓存层 3. 异步写入 |
| **事件风暴** | 高并发事件导致系统过载 | 1. 事件队列 2. 背压控制 3. 批量处理 |
| **Skills执行超时** | Skills调用超时导致工作流卡住 | 1. 超时设置 2. 重试机制 3. 降级策略 |

---

> **维护建议**：建议每周进行进度评审，每月进行技术评审。遇到重大技术决策时，召开架构评审会议。定期检查原有功能兼容性，确保升级不影响现有业务。
