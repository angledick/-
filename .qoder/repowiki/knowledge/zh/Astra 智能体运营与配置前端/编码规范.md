- API 客户端采用模块化命名空间导出（如 `agentsApi`, `skillsApi`），内部统一使用封装的 `request` 工具函数处理 Fetch 请求与鉴权头。
- 配置管理类页面（Agent/Skill/Tool）遵循统一的 CRUD 交互范式：列表页展示卡片概览，通过 Modal 组件完成详情编辑与新建操作。
- 全局状态存储（Zustand）与上下文提供者（Context）分离，前者用于高频更新的 UI 状态或运行时配置，后者用于生命周期较长的会话级数据（如用户信息、WS 连接）。
- UI 样式严格遵循 Apple-inspired 设计规范，大量使用特定的灰度色值（如 `#F5F5F7` 背景, `#1D1D1F` 文本）与圆角边框（`rounded-xl`）保持视觉一致性。
