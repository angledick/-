- **技术栈与构建**：采用 Vite + TypeScript + React 19 架构，使用 Tailwind CSS v4 进行样式管理，通过 `vite.config.ts` 配置开发代理指向后端服务（8001端口）。
- **状态管理分层**：使用 Zustand 实现轻量级全局状态管理（如 `AppStore` 管理侧边栏折叠与 Agent 运行时配置），结合 React Context 处理跨组件的复杂逻辑（如 `AuthContext` 处理 JWT 认证，`WebSocketContext` 管理实时消息推送与心跳）。
- **API 抽象层**：在 `src/api/config.ts` 中集中封装了所有 RESTful 接口调用，按业务域（Agents, Skills, Tools, OAuth, Scheduler 等）划分命名空间，统一处理鉴权头注入与错误拦截。
- **路由与布局**：基于 `react-router-dom` 实现嵌套路由，`Layout` 组件作为主框架集成侧边栏导航、顶部状态栏（WS 连接状态）及通知中心，受保护路由通过 `AuthProvider` 进行登录态校验。
- **组件化设计**：遵循“页面-组件”分离原则，`pages` 目录负责路由级业务逻辑组装，`components` 目录按功能域（config, memory, metrics）拆分可复用 UI 单元，配置类页面普遍采用“卡片列表 + 编辑模态框”的交互模式。
