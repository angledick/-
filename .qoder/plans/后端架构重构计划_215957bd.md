# 后端架构重构计划

## 总目标

1. 功能职责清晰划分，大文件按领域拆分
2. Agent/Skills/Tools/OAuth/模型配置与主架构代码彻底解耦，统一移到 `backend/data/` 下
3. 每个Agent对应独立 `.md` 配置文件，启动时自动扫描加载
4. 存储层统一、DI容器化、SDK层分层

---

## Phase 2: 配置与代码解耦（核心目标，最高优先级）

### Task 2.1: 创建 data/agents/ 目录及 Agent .md 规范

**目录结构**:
```
backend/data/agents/
  qa.md
  general.md
  export_law.md
  tax.md
  culture.md
  cert.md
  _template.md     ← 用户创建自定义Agent的模板
```

**Agent .md 格式规范**:
```markdown
---
name: QA 系统管理 Agent
type: qa
description: 系统管理智能体——权限最高、功能最全
enabled: true
sort_order: -1
sdk_config:
  enabled: true
  include_hook_events: true
skills: []
tools: ["tool_metaso_search"]
oauth_connections: []
---

你是避风港系统的「QA系统管理Agent」——拥有系统最高权限和最全功能集。

## 能力范围
... system prompt 正文 ...
```

### Task 2.2: 编写 Agent .md 加载器

**新建文件**: `backend/app/storage/agent_md_loader.py`

**职责**:
- 扫描 `data/agents/*.md`
- 解析 front-matter YAML + body (system_prompt)
- 返回统一 dict 结构
- 支持热加载 (reload_agents)
- 与现有 SQLite `agent_configs` 表互相同步（启动时加载到数据库）

### Task 2.3: 解耦内置 Agent system_prompt

**操作**:
- 将 `agent_config_store.py` 中 `DEFAULT_AGENTS` 的 186 行硬编码 system_prompt 全部移至 `data/agents/*.md`
- `agent_config_store.py` 改为纯 CRUD 代理（从 `.md` 读取或从 SQLite 读取）
- `init_default_agents()` 改为从 `data/agents/` 扫描

**涉及文件**: `app/storage/agent_config_store.py`

### Task 2.4: Skills 配置解耦到 data/skills/

**操作**:
- 新建 `data/skills/` 目录
- 将 `core/skill_registry.py` 中的内置 Skills 定义移到 `data/skills/*.md`
- 技能注册表改为从文件系统加载
- Skills 执行/推荐逻辑保留在 core 中

### Task 2.5: Tools 配置解耦到 data/tools/

**操作**:
- 新建 `data/tools/` 目录
- 将 `api/tools.py` 的内置工具定义（6个默认工具）移到 `data/tools/*.yaml`
- `api/tools.py` 仅保留 CRUD 端点，数据来源改为文件系统
- 移除 `data/config/tools.json`，统一用 `data/tools/` 目录

### Task 2.6: OAuth 配置集中化

**操作**:
- OAuth Provider 模板定义从 `core/oauth_manager.py` 移到 `data/oauth/providers.yaml`
- OAuth connections 存储路径保持 `data/config/oauth_connections.json` 但归入 `data/oauth/`
- `agent_extensions.json` 从根目录 `data/config/` 移至 `backend/data/agents/extensions.json`

### Task 2.7: 模型配置解耦

**操作**:
- 模型路由定义从 `core/model_router.py` 移到 `data/models/routes.yaml`
- `api/model_config.py` 仅保留代理层

### Task 2.8: 修复配置文件路径不一致

**操作**:
- `agent_extensions.json` 从根目录 `data/config/agent_extensions.json` → `backend/data/agents/extensions.json`
- 统一 `backend/data/` 下的配置目录结构
- 更新所有引用路径的代码

### Task 2.9: 合并事件/Worker配置加载器

**现状**: 事件配置从 `data/config/events/*.md` 读取，Worker 配置从 `data/config/workers/*.md` 读取，各写一套加载逻辑

**操作**:
- 新建 `app/storage/md_config_loader.py`: 统一 Markdown 配置加载器
- 支持 front-matter YAML 解析 + body 提取
- `worker_registry.py` 和 `event_config.py` 共用该加载器

## Phase 3: Agent .md 驱动初始化（用户核心诉求）

### Task 3.1: 实现 Agent 初始化协议

**新建文件**: `app/core/agent_initializer.py`

**职责**:
- 启动时扫描 `data/agents/*.md`
- 解析 front-matter → 获取 SDK 配置、Skills、Tools、OAuth 绑定
- 根据配置初始化对应的 Worker/工具链
- 支持热加载 (POST /admin/reload-agents)

### Task 3.2: 改造 main.py 启动流程

**现状**: 50+ 行手工初始化代码

**操作**:
- 新建 `app/core/init_sequence.py`: 定义组件初始化顺序和依赖拓扑
- 启动流程简化为：
  1. 基础设施 (DB/Chroma)
  2. 配置加载器 (Agent .md / Skills / Tools)
  3. 核心组件 (EventBus / Scheduler / WorkerRegistry)
  4. Agent初始化（从 .md 加载）
  5. SDK / OAuth / ProactiveEngine

### Task 3.3: 为每个 Agent 创建 .md 文件

**操作**:
- 为现有6个 Agent 分别创建 `data/agents/qa.md`, `general.md`, `export_law.md`, `tax.md`, `culture.md`, `cert.md`
- 每个 .md 包含完整的 SDK 配置、工具绑定、数据读取/修改方法入口
- 提供 `_template.md` 供用户创建自定义 Agent

## Phase 5: DI容器化与初始化管理（中优先级）

### Task 5.1: 引入 AppContainer

**新建文件**: `app/core/container.py`

```python
class Component:
    name: str
    dependencies: List[str]
    async def init(): ...
    async def start(): ...
    async def stop(): ...

class AppContainer:
    def register(Component): ...
    async def init_all(): ...  # 按拓扑排序
    async def start_all(): ...
    async def shutdown(): ...
```

### Task 5.2: 淘汰分散的 get_xxx() 工厂函数

**操作**:
- 将所有 `get_xxx()` 单例工厂改为 `container.get(ComponentName)`
- Startup 中只调用 `container.init_all()` + `container.start_all()`

## 风险与注意事项

1. **向后兼容**: 所有 API 端点路径不变，只做内部文件拆分
2. **测试先行**: 每个 Task 先写/更新测试，再改代码
3. **分批上线的建议顺序**: P1全部 → P2部分(2.1-2.3) → P3(3.1) → P2剩余 → P4 → P5 → P6
4. **热加载能力**: .md 驱动后，用户修改 Agent .md 无需重启服务（通过 /admin/reload-agents 触发）
