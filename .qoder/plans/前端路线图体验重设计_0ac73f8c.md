# 前端变更路线图 · 体验重设计

## 核心设计理念

**从系统功能驱动 → 业务阶段驱动 + 对话式配置**

---

## 页面架构重设计

```
侧边栏导航:
├── 概览 (Overview) — 全局合规健康度 + 事件流
├── 产品工作台 (Product Workspace)
│   ├── 产品卡片列表 — 合规状态可视化
│   └── 产品详情 → 流式对话工作台
├── 配置中心 (Configuration Hub)
│   ├── Agent配置 — 选择关联的Skills/Tools/OAuth
│   ├── Skills管理 — CRUD + GitHub链接/ZIP上传/MD撰写
│   ├── Tools管理 — CRUD + 后端联动
│   └── OAuth管理 — CRUD + 对接系统配置
├── 系统合规 (System Compliance)
│   └── 全局合规展示 — 事件时间线 + 风险矩阵
└── 对话中心 (Chat Hub) — 流式对话 + 事件可视化
```

---

## 核心功能模块

### 1. 流式对话交互系统
- **流式输出**: SSE/WebSocket实现打字机效果
- **事件流可视化**: Skill事件、思考过程、对话事件、执行计划实时展示
- **产品级对话**: 产品卡片点击进入专属对话上下文
- **对话配置**: 通过自然语言对话完成系统对接配置

### 2. 配置中心 (Configuration Hub)

#### Agent配置卡片
- 选择关联的Skills列表
- 选择关联的Tools列表  
- 选择关联的OAuth对接
- 配置变更实时预览
- 后端联动保存

#### Skills/Tools/OAuth管理卡片
- **卡片展示**: 网格/列表视图切换
- **CRUD操作**: 创建、读取、更新、删除
- **配置来源**:
  - GitHub链接导入（SDK自动配置）
  - ZIP文件上传
  - 手动撰写变更MD
- **文件刷新**: 读取文件变更并刷新配置状态

### 3. 产品合规工作台
- 产品卡片展示合规状态
- 点击进入产品专属对话
- 对话中流式展示:
  - Skill执行事件
  - AI思考过程
  - 对话事件流
  - 执行计划进度

### 4. 系统合规全局展示
- 全局事件时间线
- 合规风险矩阵
- 实时告警推送

---

## 技术实现要点

### 流式对话
```
SSE连接 → 事件解析 → 分类渲染:
├── skill_event (技能执行)
├── thinking (思考过程)
├── chat_message (对话消息)
├── plan_update (计划更新)
└── tool_call (工具调用)
```

### 配置管理
```
配置卡片 → 后端API联动:
├── /api/agents/{id}/skills
├── /api/agents/{id}/tools
├── /api/agents/{id}/oauth
├── /api/skills (CRUD)
├── /api/tools (CRUD)
└── /api/oauth (CRUD)
```

### 文件导入流程
```
GitHub链接 → SDK解析 → 配置生成
ZIP上传 → 解压 → 配置提取
MD撰写 → 解析 → 配置应用
```
