# 远程 main 功能整合到 v4 新架构

## 差异总览

| 维度 | origin/main (旧架构) | 本地 dev/roadmap-v4 (新架构) |
|------|---------------------|---------------------------|
| SDK底座 | codex-client (Codex CLI) | claude-agent-sdk (Anthropic 官方) |
| 配置模型 | Settings: openrouter + codex | Settings: Claude SDK 全参数 + metaso |
| 对话入口 | `api/chat.py` (同步) | `api/chat_stream.py` (SSE流式) |
| Agent模型 | `CodexAgent` (codex-client) | `AstraAssistant` (claude-agent-sdk) |
| 服务层 | codex_agent.py + codex_tools.py | astra_assistant.py + astra_tools.py |
| 事件体系 | 无 EventBus | 完整 EventBus + Worker Registry |
| 启动流程 | 简单 (init_admin + scheduler) | 9阶段渐进式启动 |

## 需要迁移的功能模块（4个）

### 模块 A: RAG 知识库导入（PDF/URL）
- `backend/app/api/knowledge_import.py` — 知识库上传/URL导入/搜索 API
- `backend/app/services/doc_processor.py` — PDF/URL 内容提取 + 分块
- `backend/app/storage/knowledge_doc_store.py` — 文档元数据 SQLite 存储
- 依赖: pypdf, beautifulsoup4 (requirements.txt 需补充)

### 模块 B: OpenCLI 浏览器控制
- `backend/app/api/browser.py` — OpenCLI API 路由
- `backend/app/services/opencli_service.py` — subprocess/daemon HTTP 封装

### 模块 C: 新闻监控（采集 + AI分析）
- `backend/app/api/news_monitor.py` — 新闻列表/摘要/采集/关键词 API
- `backend/app/services/news_collector.py` — 多数据源 RSS/HTTP 采集器
- `backend/app/services/news_analyzer.py` — LLM 风险方向分析
- `backend/app/storage/news_store.py` — 新闻 + 分析结果 SQLite 存储
- 依赖: feedparser (requirements.txt 需补充)

### 模块 D: 预警推送（飞书/企微）
- `backend/app/api/notify_config.py` — 通知渠道 CRUD API
- `backend/app/services/notifier.py` — 飞书/企微 Webhook 推送
- `backend/app/storage/notify_config_store.py` — 渠道配置 SQLite 存储

### Task 3: 迁移服务层 (services/)

#### 3.1 直接复制（无 Codex 依赖）：
- `backend/app/services/doc_processor.py` — PDF/URL 处理器
- `backend/app/services/opencli_service.py` — OpenCLI subprocess + daemon 封装
- `backend/app/services/notifier.py` — 飞书/企微 Webhook

#### 3.2 需适配的文件：
- `backend/app/services/news_analyzer.py`
  - 原版使用 `settings.active_llm_api_key` / `settings.active_llm_base_url`
  - 适配方案：新架构没有 `active_llm_*` 属性，需要：
    - 方案 A（推荐）：在 `config.py` 中增加兼容属性 `active_llm_api_key` / `active_llm_base_url` / `llm_model`（映射到现有 LLM 配置或新增字段）
    - 方案 B：直接修改 news_analyzer.py 使用 claude-agent-sdk 调用
  - **推荐方案 A**：在 Settings 类中添加如下兼容字段：
    ```python
    llm_api_key: str = ""
    llm_base_url: str = ""
    llm_model: str = "mimo-v2.5-pro"
    llm_disable_thinking: bool = True
    
    @property
    def active_llm_api_key(self) -> str:
        return self.llm_api_key or self.anthropic_api_key
    
    @property
    def active_llm_base_url(self) -> str:
        return self.llm_base_url or "https://openrouter.ai/api/v1"
    ```

- `backend/app/services/news_collector.py`
  - 纯 HTTP 采集，无 SDK 依赖，可直接复制
  - 仅依赖 `app.storage.news_store.get_keywords`（Task 2 已迁入）

### Task 6: 注册路由到 main.py

在 `backend/app/main.py` 中添加新路由导入和注册：

```python
# 远程功能整合：知识库导入 + OpenCLI + 新闻监控 + 预警推送
from app.api import knowledge_import as knowledge_import_router
from app.api import browser as browser_router
from app.api import news_monitor as news_monitor_router
from app.api import notify_config as notify_config_router

# 注册到 app
app.include_router(knowledge_import_router.router)
app.include_router(browser_router.router)
app.include_router(news_monitor_router.router)
app.include_router(notify_config_router.router)
```
