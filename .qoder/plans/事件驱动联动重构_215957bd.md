# 事件驱动联动重构

## 架构目标

```
event_bus.publish(event)
       |
       v
 ManagerAgent.on_event()  <-- bus.on_all 全局监听
       |
       +---> WSManager.send_to_user()    [session_update / new_message / alert / scan_update]
       +---> WorkerRegistry.dispatch()   [related_worker 自动调度]
       +---> metrics.json 更新           [product_id 存在时自动聚合]
```

角色分工:
- **ManagerAgent**: SDK 中枢，事件 -> WS推送 + Worker调度 + 指标更新
- **QAAgent**: 配置目录 CRUD（事件/Worker/规则/通知的注册/修改/删除）
- **Sub-Workers**: 事件响应，按需求决定是否创建定时任务/执行工作流
- **WSManager**: WebSocket 连接管理 + 按任务分配上下文

## Task 1: WSManager 统一推送接口

文件: `backend/app/services/ws_manager.py`

新增 `send_to_user(user_id, message)` 通用方法，支持前端需求文档的 `{type, payload}` 格式:

```python
async def send_to_user(self, user_id: str, message: dict):
    """向指定用户推送任意格式消息（通用接口）"""
    if user_id not in self._connections:
        return
    text = json.dumps(message, ensure_ascii=False)
    dead = []
    for ws in self._connections[user_id]:
        try:
            await ws.send_text(text)
        except Exception:
            dead.append(ws)
    for ws in dead:
        await self.disconnect(user_id, ws)
```

保留 `send_alert` / `send_scan_update` / `broadcast` 作为便捷方法，内部委托给 `send_to_user`。

## Task 2: ManagerAgent 事件驱动中枢

文件: `backend/app/core/manager_agent.py`

新增 `on_event(event: EventRecord)` 方法，作为 event_bus 全局处理器:

```python
async def on_event(self, event: EventRecord):
    """事件驱动中枢 — 接收所有事件，路由到 WS推送 / Worker调度 / 指标更新"""
    # 1. WebSocket 推送
    await self._route_to_websocket(event)
    # 2. Worker 调度（如果事件定义有 related_worker）
    await self._route_to_worker(event)
    # 3. 指标更新（如果事件关联 product_id）
    await self._route_to_metrics(event)
```

**事件类型 -> WS 消息映射表**:

| 事件类型前缀 | WS 消息 type | payload |
|-------------|-------------|---------|
| `session:created` / `session:deleted` | `session_update` | `{}` |
| `message:created` | `new_message` | `{session_id, message}` |
| severity >= high 或 `risk:` | `alert` | 事件数据 |
| `scan:` | `scan_update` | `{status, progress}` |

**Worker 调度逻辑**: 从 event_registry 查找 EventDefinition，若 `related_worker` 非空则调用 `worker_registry.execute_worker_task()`。

**指标更新逻辑**: 若事件有 `product_id`，更新 `data/products/{id}/metrics.json` 中的事件计数。
