## 一、系统架构

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    Frontend (React SPA)                    │
│                                                           │
│  Pages(31) → Context(Auth/WS/Notif) → API Client Layer   │
│                                                           │
│  TanStack Query 数据缓存 + authFetch 认证注入              │
└────────────────────────┬────────────────────────────────┘
                         │ HTTP / SSE / WebSocket
                         │
┌────────────────────────┴────────────────────────────────┐
│                  Backend (FastAPI + Python)                │
│                                                           │
│  ┌─────────────┐  ┌──────────────┐  ┌─────────────────┐  │
│  │ Middleware   │  │ 50 Routers   │  │ Core Engine     │  │
│  │ CORS + JWT   │→ │ 200+ 端点    │→ │ (单例模式)       │  │
│  └─────────────┘  └──────────────┘  └────────┬────────┘  │
│                                              │            │
│  ┌───────────────────────────────────────────┴─────────┐ │
│  │              Storage Layer                           │ │
│  │  SQLite │ JSON │ Markdown │ ChromaDB │ FileSystem   │ │
│  └─────────────────────────────────────────────────────┘ │
└───────────────────────────────────────────────────────────┘
                         │
          ┌──────────────┼──────────────┐
          │              │              │
    ┌─────┴─────┐  ┌─────┴─────┐  ┌────┴────┐
    │  Shopify  │  │   飞书     │  │ OpenRouter│
    │ Admin API │  │   Bot API  │  │ / Claude  │
    └───────────┘  └───────────┘  └──────────┘
```

### 1.2 核心设计原则

| 原则 | 实现 |
|------|------|
| **事件驱动** | 所有业务操作通过 EventBus 发布/订阅，组件松耦合 |
| **单例模式** | Core Engine 层所有组件使用全局单例 + get_xxx() 懒加载 |
| **配置驱动** | Agent/Worker/Event/Skill 均由 YAML/JSON/MD 文件定义，热加载 |
| **分层存储** | SQLite（结构化）+ JSON（配置）+ ChromaDB（向量）+ Markdown（知识） |
| **渐进增强** | Phase 1 → 4 逐步叠加功能，每个 Phase 独立可用 |

## 二、后端架构

### 2.1 启动流程（5 Phase）

```
lifespan()
  │
  ├─ Phase 1: 基础设施
  │   ├─ init_admin_if_empty()          # 初始化管理员
  │   ├─ EventBus + EventRegistry       # 事件总线
  │   ├─ WorkerRegistry                 # Worker 注册表
  │   └─ AgentInitializer               # Agent 文件加载
  │
  ├─ Phase 2: 核心组件
  │   ├─ QAAgent → 绑定 Registries
  │   ├─ ManagerAgent → 绑定 EventBus (on_all)
  │   └─ Claude Agent SDK 预加载
  │
  ├─ Phase 3: 扩展组件
  │   ├─ OAuthManager / AutoPullEngine
  │   ├─ ChannelAdapter / SecuritySandbox
  │   ├─ SkillRegistry / PluginManager
  │   ├─ RBAC / ApprovalEngine / OperationGuard
  │   ├─ ProactiveEngine
  │   └─ RiskIntelEngine → 恢复周期任务
  │
  ├─ Phase 4: 调度器启动
  │   └─ APScheduler (AsyncIOScheduler)
  │       ├─ 风险情报检索（每 1 小时）
  │       ├─ 心跳自检（每 5 分钟）
  │       └─ Shopify 商品同步（每 20 分钟）
  │
  └─ Phase 5: 外部监听器
      ├─ FeishuListener → 注册到 Dispatcher
      └─ ShopifyEventListener → 定时同步 + Webhook
```

## 三、事件驱动架构

### 3.1 事件流

```
事件源                    EventBus                    处理者
─────────                ────────                    ──────
Shopify Webhook    ──→                            ──→  Compliance Worker
用户操作           ──→   bus.emit(event)    ──→   ──→  Regulation Worker
定时任务           ──→   bus.on_all(handler)──→   ──→  Listing Worker
飞书消息           ──→                            ──→  Manager Agent
                                            │
                                            └──→  WebSocket 推送前端
```

### 3.2 事件定义示例

```yaml
# data/events/builtin/shopify_events.md
event_code: product:created
source: shopify
description: Shopify 产品创建
fields:
  - product_id: 产品 ID
  - product_title: 产品标题（合规审查核心）
  - product_type: 产品类型（HS 编码查询）
  - compliance_query: 已拼接的合规查询文本
```

### 3.3 六步执行流水线

```
感知(Perceive) → 检查(Check) → 推荐(Recommend) → 告知(Notify) → 交互(Interact) → 处理(Process)
    │               │               │                │               │               │
  事件监听        合规规则引擎      AI 推荐         多渠道推送      用户确认/修改   自动执行
  Webhook         HS/VAT/认证      LLM 分析        飞书/Webhook    前端交互        Worker 调度
```

## 四、Shopify 集成架构

### 4.1 集成链路

```
Shopify Admin API
       │
       ▼
shopify_api.py (sync_to_local)
       │
       ├── OAuth 授权 → shopify_oauth_flow.py
       ├── 产品同步 → ProductStorage (JSON)
       ├── 缺失检测 → FeishuClient.send_message()
       └── Webhook 监听 → EventBus → ManagerAgent
```

### 4.2 定时同步流程

```python
# scheduler.py — _run_shopify_product_sync()
1. sync_to_local(limit=250)       # 从 Shopify 拉取产品
2. 对比本地 ProductStorage        # 检测缺失商品
3. 如果有缺失:
   - 飞书 Bot 推送缺失商品列表
   - 格式: [shopify_id] 标题 (类型) — 状态
4. 记录同步日志
```

## 六、安全设计

### 6.1 认证与授权

```
JWT Token Flow:
  Login → JWT(HS256) → 前端 localStorage → authFetch 注入 Authorization Header

RBAC 权限:
  User → Role → Permission → Operation Guard 拦截

审批流:
  高危操作 → ApprovalEngine → 审批通过 → 执行
```

### 6.2 安全沙箱

- `SecuritySandbox`: Worker 代码执行隔离
- Webhook HMAC-SHA256 签名验证
- `webhook_event_log` 幂等去重

## 七、性能与扩展

### 7.1 数据缓存

| 层级 | 机制 |
|------|------|
| 前端 | TanStack Query 自动缓存 + 失效 |
| 后端 | Core Engine 单例缓存（全局唯一实例） |
| 存储 | JSON 文件全量加载到内存 + 定时持久化 |

### 7.2 实时通信

| 通道 | 协议 | 场景 |
|------|------|------|
| AI 对话 | SSE | 流式生成响应 |
| 事件推送 | WebSocket | 风险预警、物流更新、会话变更 |
| Webhook | HTTP POST | Shopify/17TRACK/AfterShip 回调 |

### 7.3 扩展点

| 扩展类型 | 方式 |
|----------|------|
| 新增合规规则 | 编辑 `compliance_rules.py` + 添加规则数据 |
| 新增 Agent | 在 `data/agents/` 添加 `.md` 定义文件 |
| 新增 Worker | 在 `data/workers/` 添加 `.md` 定义文件 |
| 新增 Skill | 在 `data/skills/` 添加 `SKILL.md` |
| 新增事件 | 在 `data/events/builtin/` 添加 `.md` |
| 新增通知渠道 | 实现 `ChannelAdapter` 接口 |
