## 一、系统架构全景

### 1.1 设计哲学

| 原则 | 说明 |
|------|------|
| 事件驱动，主动感知 | 不等用户提问，系统主动感知合规热点并推送 |
| 记忆优先，上下文完整 | 所有数据规范化为记忆树，Agent每天已拥有全天上下文 |
| 多Agent协作，职责隔离 | Manager拆解任务，Worker隔离执行，群聊式调度 |
| Token经济，成本可控 | 所有工具输出经TokenJuice压缩，降低80% token消耗 |
| 本地优先，数据主权 | 记忆/配置/密钥存储在本地，加密不动 |
| OS级组件化 | 像操作系统一样提供进程管理、文件系统、权限控制、网络栈 |

### 3.2 配置加载原则

- **零硬编码**：所有系统配置由 `data/` 目录下的结构化文件（`.yaml`/`.md`/`.json`）定义和加载
- **无回退**：配置文件必须存在且可解析，否则直接抛异常（`FileNotFoundError` / `ValueError`）
- **热生效**：REST API 操作即时写入文件系统，下次使用即生效

### 3.3 Tools 注册与消费链路（v3.2 修正）

## 四、重要功能详解

### 4.1 事件驱动六阶段合规流水线

```
感知(Sense) → 检查(Check) → 推荐(Recommend) → 告知(Notify) → 交互(Interact) → 处理(Process)
```

| 阶段 | 执行者 | 输入 | 输出 |
|------|--------|------|------|
| 感知 | EventBus + MarketMonitor | 用户动作/外部API/定时调度/规则引擎 | 标准化EventRecord |
| 检查 | Worker + Skill + RuleEngine | EventRecord | 检查结果 + 风险等级 |
| 推荐 | Skill推荐器 + 规则引擎 | 检查结果 | 含置信度的操作建议列表 |
| 告知 | 通知引擎 + 频道适配器 | 推荐结果 | 多端消息推送 |
| 交互 | SSE对话引擎 | 用户响应 | 执行指令 |
| 处理 | Workflow + Skill流 | 确认指令 | 操作结果 + MCP回写 |

### 4.2 多Agent调度（聊天室模式）

```
Manager Agent (协调者)
  │  任务拆解 + 分配 + 监督
  │  读取全局记忆：活跃Agent列表 + 各Agent事件流
  ├── compliance_worker    — 合规检查（全阶段）
  ├── certification_worker  — 认证管理（全阶段）
  ├── store_worker          — 建站配置（阶段1）
  ├── product_worker        — 选品设计（阶段2）
  ├── supplier_worker       — 供应商审核（阶段3）
  ├── listing_worker        — 商品上架（阶段4）
  ├── payment_worker        — 支付配置（阶段5）
  ├── order_worker          — 订单处理（阶段6）
  ├── customs_worker        — 出口报关（阶段7）
  ├── logistics_worker      — 物流配送（阶段8）
  ├── aftersale_worker      — 售后退货（阶段9）
  ├── finance_worker        — 财务结算（阶段10）
  └── monitoring_worker     — 指标监控（全阶段）
```

### 4.3 SSE流式对话协议

```typescript
type StreamEvent =
  | { type: 'token'; content: string }
  | { type: 'skill_start'; skill: string; args: object }
  | { type: 'skill_end'; skill: string; result: object; duration_ms?: number; status: 'success'|'error' }
  | { type: 'thinking'; content: string; depth?: number }
  | { type: 'plan'; steps: PlanStep[]; current: number }
  | { type: 'action_card'; actions: Action[] }
  | { type: 'error'; code: string; message: string; recoverable?: boolean }
  | { type: 'done'; finish_reason?: string; usage?: object }
```

### 4.4 记忆树（4层级摘要）

```
L3 全局索引（≤500 token/产品） — 所有领域概览的索引
  └── L2 领域概览（≤1k token） — 德国市场合规全景 / LED产品线全景
       └── L1 话题摘要（≤2k token） — WEEE认证到期 / CE标志检查
            └── L0 原始片段（≤3k token） — 合规记录 / 事件条目 / 人工备注
```

### 4.5 8类事件体系

| 事件类别 | 编码 | 作用域 | 子类示例 |
|----------|------|--------|----------|
| 生命周期事件 | `lifecycle` | 产品级 | product:created / status_changed / ended |
| 合规检查事件 | `compliance` | 产品级 | check_started / check_passed / check_failed |
| 认证管理事件 | `certification` | 产品级 | uploaded / expiring / expired |
| 订单事件 | `order` | 产品级 | created / shipped / returned |
| 市场法规事件 | `regulation` | 全局 | updated / new / repealed |
| 风险预警事件 | `risk_alert` | 全局/产品 | new / resolved / escalated |
| 系统事件 | `system` | 全局 | config_changed / error / health_check |
| 用户操作事件 | `user_action` | 全局 | login / config_modified / approved |

### 4.6 事件订阅机制（4种模式）

| 订阅类型 | 说明 | 适用场景 |
|----------|------|----------|
| 精准订阅 | 特定产品+特定事件类型 | 单产品合规结果监听 |
| 批量订阅 | 一组产品的特定事件 | 批次产品认证到期监控 |
| 全局订阅 | 全部产品的某类事件 | 全局法规变更通知 |
| 条件订阅 | 按字段条件过滤(severity=critical) | 仅高风险事件告警 |

### 4.7 四大支柱协作模型

```
MCP（数据层）── 事件感知 + 结果回写
  ↓
Skill（规则层）── 合规检查 + 操作推荐
  ↓
Workflow（确定性层）── 步骤编排 + 状态追踪 + 回滚
  ↑
Cowork（入口层）── SSE对话 + 用户确认 + 操作卡片
```

## 八、数据流转

### 8.1 产品全生命周期数据流

```
外部数据源                     系统内部                          存储层
┌──────────┐         ┌──────────────────┐          ┌──────────────────┐
│Shopify   │──OAuth──→│ AutoPullEngine   │──压缩──→ │ products/{id}/   │
│Webhook   │         │ (20min周期拉取)    │          │   events/        │
│API       │         └────────┬─────────┘          │   metrics/       │
└──────────┘                  │                     │   memory/        │
                              ▼                     │   knowledge/     │
┌──────────┐         ┌──────────────────┐          └──────────────────┘
│法规数据源 │──扫描──→│ MarketMonitor    │──事件──→ global/events/
│(RSS/API) │         │ (每6h法规扫描)    │          global/knowledge/
└──────────┘         └──────────────────┘
                              │
                              ▼
┌──────────┐         ┌──────────────────┐          ┌──────────────────┐
│用户操作   │──SSE───→│ 全局事件总线      │──路由──→ │ DataRouter       │
│(前端对话) │         │ (EventBus)       │          │ 按事件类型分发    │
└──────────┘         └──────────────────┘          └──────────────────┘
```

### 8.2 热点事件 → 数据变更映射

| 热点事件 | 触发变更 | 影响范围 |
|----------|----------|----------|
| `product:created` | events追加 + metadata初始化 + global/memory活跃快照更新 | 单产品 |
| `compliance:check_failed` | events + memory/compliance + memory/issues + metrics下降 | 单产品+全局 |
| `certification:expiring` | events + metrics(status=warning) + memory/compliance | 单产品+全局 |
| `certification:expired` | events + metrics(status=critical) + memory/issues续期整改 | 单产品+全局 |
| `regulation:updated` | product/memory评估 + global/events + global/knowledge增量 | 涉市全部产品 |
| `order:created` | events + metrics(order_count) + global/metrics | 单产品+全局 |
| `risk:metric_alert` | events + metrics状态更新 + memory/issues风险整改 | 单产品 |

### EventRecord

```typescript
interface EventRecord {
  id: string;
  type: string;              // 如 'compliance:check_failed'
  category: string;          // 8类之一
  scope: 'product' | 'global';
  product_id?: string;
  data: Record<string, any>;
  data_sources?: { read: string[]; write: string[] };
  created_at: string;
  severity?: 'low' | 'medium' | 'high' | 'critical';
}
```

### ProductLifecycleStage

```
concept | design | sourcing | ready | active | fulfilling | aftersale | end
```

### NotificationPayload

```typescript
interface NotificationPayload {
  id: string;
  type: 'compliance_check' | 'certification_expiry' | 'risk_alert' | 'regulation_change' | 'system_notice';
  title: string;
  message: string;
  product_id: string;     // 深度链接跳转目标
  stage: string;          // 业务阶段
  severity: 'low' | 'medium' | 'high' | 'critical';
  created_at: string;
  is_read: boolean;
}
```

### Tool (来自 _registry.yaml)

```typescript
interface ToolDefinition {
  id: string;               // 如 'tool_metaso_search'
  name: string;             // 中文名称
  description: string;      // 功能描述
  tool_type: 'builtin' | 'script';
  category: 'compliance' | 'search' | 'browser' | 'monitor';
  enabled: boolean;
  script?: string;          // script 类型：指向 impl/ 下的可执行文件
  script_args?: string[];   // 命令行参数模板
}
```
