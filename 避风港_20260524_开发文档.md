## 一、项目全景

### 1.1 产品定位

避风港 是一款面向中小出海企业的低成本、全链路、可解释的 AI 合规基础设施。核心使命是将传统高昂的合规服务转化为普惠型数字化解决方案。

- **技术栈**: FastAPI + React + ChromaDB + LLM (OpenRouter) + Codex SDK
- **架构模式**: 规则引擎 + LLM 混合 + Codex Agent 协同
- **存储架构**: L0-L5 分层文件存储
- **AI 引擎**: Codex CLI 驱动的 Agent 编排（替代传统多智能体框架）

### 1.2 三阶段产品生命周期模型

```
产品事前（Pre）            产品发布事中（Mid）         产品发布售后（Post）
─────────────────────────────────────────────────────────────────────

产品风控Agent               热点事件预警               处置方案咨询
（专利·风俗文化）            Codex联网搜索             （法律·文化）

法务Agent                   物流Agent
（法规·认证）                （清关协调）

相关变化预警
Codex市场监控
```

| 阶段 | 覆盖场景 | 当前状态 |
|------|---------|---------|
| **事前 Pre** | 新品合规评估、专利风险、文化禁忌、认证预检 | ⚠️ 部分 |
| **事中 Mid** | 市场热点监控、清关物流协调、实时预警 | ⚠️ 部分 |
| **售后 Post** | 违规处置方案、法律咨询、合规整改 | ❌ 未开始 |

### 3.4 Skill 扩展机制说明

每个 Agent 可通过编写 Codex Skill 文件实现，Skill 文件存储在 `data/skills/` 目录，格式为 Markdown：

```markdown
---
name: product_risk
description: 产品风控检查（专利·文化·安全）
---

你是产品风控 Agent。你需要：
1. 查询专利数据库检查侵权风险
2. 检查目标市场的文化/宗教禁忌
3. 输出风险等级与建议

可用工具：
- lookup_hs_code: HS 编码查询
- mcp_patent_search: 专利检索（待集成）
- mcp_cultural_db: 文化禁忌知识库（待构建）
```

优势：
- **无需改代码**：新增/修改 Skill 只需增删 Markdown 文件
- **热加载生效**：修改后调用 `POST /api/v1/prompts/reload` 即时生效
- **渐进式交付**：可先写简单 Skill 指令，再逐步补充知识库和 MCP 工具
- **Codex 原生支持**：Skills 是 Codex CLI 的一等公民，支持联网搜索、CLI、工具调用

## 七、关键决策记录

| 决策 | 内容 | 原因 |
|------|------|------|
| 使用 Codex SDK | 替代自建多智能体框架 | 避免风控、利用联网/CLI全能力 |
| JWT 认证 | 采用 JWT Bearer Token + bcrypt | 轻量级认证，无需数据库级多租户 |
| SQLite 会话存储 | 替代 SQLAlchemy ORM | 轻量文件数据库，降低依赖 |
| 多 Agent 配置存储 | SQLite 持久化 Agent 预设 | 支持运行时 CRUD，无需改代码 |
| LLM 配置热切换 | 激活时同步全局 settings | 前端可切换 LLM Provider，即时生效 |
| Prompt 文件配置 | 所有 prompt 存 YAML，支持热加载 | 便于微调，不改代码 |
| Agent 即 Skill | 每个 Agent 作为 Codex Skill 文件实现 | 无需写 Python Agent 基类，增删 Skill 即增删 Agent |
| 文件级隔离 | 不引入数据库级多租户 | 初期降低架构复杂度 |
| 公开 API 优先 | 不自己写爬虫 | 降低法律风险和维护成本 |
| 安全 L1 起步 | JWT + 角色鉴权 + 密码哈希 | 避免过度工程化 |
| 区块链不实现 | 日志+数字签名替代 | 目标用户无此需求 |
| 风险监控用 Codex | 委托 Codex 联网搜索 | 替代 RSS/爬虫/自建采集 |
