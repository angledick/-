## 消息格式规范

### 统一消息结构

所有 WebSocket 消息**必须**使用以下 JSON 格式：

```typescript
{
  "type": string,       // 消息类型（必需）
  "payload": object     // 消息内容（必需，可为空对象 {}）
}
```

**示例**：
```json
{
  "type": "session_update",
  "payload": {}
}
```

**错误示例**（❌ 不要这样）：
```json
// ❌ 缺少 type 字段
{"data": {...}}

// ❌ payload 不是对象
{"type": "alert", "payload": "string"}

// ❌ 直接发送字符串
"session updated"
```

## WebSocket 管理器建议实现

### 基础结构

```python
# backend/app/core/websocket_manager.py

from typing import Dict, Set
from fastapi import WebSocket

class WebSocketManager:
    def __init__(self):
        # user_id -> Set[WebSocket]
        self.active_connections: Dict[str, Set[WebSocket]] = {}
    
    async def connect(self, websocket: WebSocket, user_id: str):
        """用户连接"""
        await websocket.accept()
        if user_id not in self.active_connections:
            self.active_connections[user_id] = set()
        self.active_connections[user_id].add(websocket)
        print(f"[WebSocket] User {user_id} connected")
    
    def disconnect(self, websocket: WebSocket, user_id: str):
        """用户断开连接"""
        if user_id in self.active_connections:
            self.active_connections[user_id].discard(websocket)
            if not self.active_connections[user_id]:
                del self.active_connections[user_id]
        print(f"[WebSocket] User {user_id} disconnected")
    
    async def send_to_user(self, user_id: str, message: dict):
        """发送消息给指定用户的所有连接"""
        if user_id not in self.active_connections:
            return
        
        disconnected = set()
        for websocket in self.active_connections[user_id]:
            try:
                await websocket.send_json(message)
            except Exception as e:
                print(f"[WebSocket] Failed to send to {user_id}: {e}")
                disconnected.add(websocket)
        
        # 清理失败的连接
        for ws in disconnected:
            self.disconnect(ws, user_id)
    
    async def broadcast(self, message: dict):
        """广播消息给所有用户"""
        for user_id in list(self.active_connections.keys()):
            await self.send_to_user(user_id, message)

# 全局单例
websocket_manager = WebSocketManager()
```

### WebSocket 端点

```python
# backend/app/api/v1/websocket.py

from fastapi import APIRouter, WebSocket, WebSocketDisconnect
from app.core.websocket_manager import websocket_manager

router = APIRouter()

@router.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket, user_id: str = "default"):
    await websocket_manager.connect(websocket, user_id)
    
    try:
        while True:
            # 接收消息（如需要）
            data = await websocket.receive_text()
            
            # 可选：处理前端发来的消息（如 ping）
            try:
                msg = json.loads(data)
                if msg.get("type") == "ping":
                    await websocket.send_json({"type": "pong", "payload": {}})
            except:
                pass
    
    except WebSocketDisconnect:
        websocket_manager.disconnect(websocket, user_id)
```

## 常见问题（FAQ）

### Q1: WebSocket 连接失败怎么办？

**A**: 检查以下几点：
1. 后端 WebSocket 服务是否启动（`:8000/api/v1/ws`）
2. CORS 配置是否允许 WebSocket 连接
3. 防火墙是否拦截 WebSocket 端口
4. 浏览器控制台是否有错误日志

### Q2: 消息格式错误会怎样？

**A**: 前端会忽略无法解析的消息，并在控制台打印错误：
```
[WebSocket] Failed to parse message: ...
```

### Q3: 如何调试 WebSocket 消息？

**A**: 
1. **后端**：在 `send_to_user` 中添加日志
   ```python
   print(f"[WebSocket] Sending to {user_id}: {message}")
   ```

2. **前端**：打开浏览器控制台，查看日志
   ```
   [WebSocket] Connected
   [WebSocket] Session list invalidated
   ```

3. **抓包**：使用 Wireshark 或浏览器 DevTools Network 面板

### Q4: 用户有多个设备连接怎么办？

**A**: `WebSocketManager` 已支持一个用户多个连接：
- `active_connections` 是 `Dict[str, Set[WebSocket]]`
- `send_to_user` 会发送给该用户的所有连接

### Q5: WebSocket 连接数过多会有性能问题吗？

**A**: 
- WebSocket 是轻量级协议，每个连接占用内存很小（~1-2 KB）
- 1000 个并发连接占用约 1-2 MB 内存
- 建议：超过 10,000 连接时考虑使用 Redis Pub/Sub 或消息队列

### Q6: 前端已经用 TanStack Query 了，为什么还需要 WebSocket？

**A**: 
- TanStack Query 是轮询或手动触发
- WebSocket 是服务端主动推送，实时性更好
- 两者结合：WebSocket 失效缓存，TanStack Query 自动重新获取
